Git Notes

Version control

Version control is a way of managing changes to codebase. It enables developers to:

  • Switch between different versions of the code (e.g. for different features)
  • See how the codebase has changed over time, and the reasons for those changes
  • Make collaborative decisions over which changes to keep
  • Rollback to an older version, if something breaks

Git

Git is a free, open source version control system. It's been the most widely-used version control system for many years.

It was developed by Linus Torvalds, who named the project after himself 🤣.

To commit changes to a github repository using GitHub Desktop:

  1. Select the changes to commit
  2. Write a description
  3. Click the "commit" button

GitHub

GitHub is a website that hosts git repositories. It makes it easy for developers to collaborate on code using git.

There are several good alternatives, including GitLab and Bitbucket.
